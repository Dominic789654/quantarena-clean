## Why

Phase 4 step 31 (docs/refactor_program_plan.md) — the finale of the report_agent decomposition program. After the previous seven extractions (`retry.py`, `chart_sanitizer.py`, `structured_report.py`, `citations.py`, `ticker_utils.py`, `forecast_requests.py`, `chart_renderer.py`, `clustering.py`), `deepear/src/agents/report_agent.py` (731 lines) contains nothing but the `ReportAgent` class itself: `__init__`, `_get_forecast_agent`, `generate_report`, `_incremental_edit`, and ~15 same-named delegator methods to the seven already-extracted modules. This step moves that remaining class into `deepear/src/agents/report/agent.py` (completing the package every other piece already lives under) and collapses `deepear/src/agents/report_agent.py` into a thin compatibility shim, mirroring how Phase 2 finished `run.py` and Phase 3 finished `backtest/workflow_adapter.py`.

**Patch-seam audit.** `git grep -n "report_agent" tests/ deepear/ backtest/ deepfund/ shared/ | grep -v "agents/report/"` finds three categories of reference: (a) plain imports — every test file and `deepear/src/agents/__init__.py` do `from deepear.src.agents.report_agent import ReportAgent`; no file imports any other name from `report_agent` (confirmed by diffing `dir(deepear.src.agents.report_agent)` before this change against every `from ... import <name>` and `<module>.<name>` grep hit — only `ReportAgent` is ever referenced this way); resolved by a plain re-export. (b) Module-attribute monkeypatches, all inside `tests/report_agent_harness.py`'s `make_report_agent` (the single construction point every report test uses): `monkeypatch.setattr(report_agent_module, "Agent", ...)` and `monkeypatch.setattr(report_agent_module, "ForecastAgent", ...)`, where `report_agent_module` is `import deepear.src.agents.report_agent as report_agent_module`. (c) Two more module-attribute patches outside the harness: `tests/test_report_agent_characterization.py`'s `test_tool_model_defaults_to_model_when_omitted` (`monkeypatch.setattr("deepear.src.agents.report_agent.Agent", ...)`, a string-path patch) and `tests/test_report_agent_citations.py`'s `test_non_incremental_report_generation_normalizes_citations` (same string-path patch on `Agent`), plus `tests/test_report_clustering.py`'s `test_delegator_forwards_self_planner_by_reference` (`monkeypatch.setattr(report_agent_module, "_cluster_signals_impl", ...)`) and `tests/test_report_chart_renderer.py`'s two `import deepear.src.agents.report_agent as report_agent_module` sites that read/patch `report_agent_module.Agent` to prove the `agent_cls` call-time-resolution decision.

Category (b)/(c) is the make-or-break case: once `ReportAgent` moves, its methods execute in `deepear.src.agents.report.agent`'s namespace, so `Agent`, `ForecastAgent`, and `_cluster_signals_impl` are looked up there at call time. A patch applied to the old `deepear.src.agents.report_agent` module would silently stop intercepting (verified experimentally — see design.md) — there is no `loguru`-singleton-style escape hatch here (`Agent`/`ForecastAgent`/`_cluster_signals_impl` are plain module-global names, not shared mutable objects). The honest fix, per the plan's "Preferred approach", is to move every one of these patch targets to the new namespace rather than leave them silently broken.

## What Changes

- Add `deepear/src/agents/report/agent.py`: move `ReportAgent` verbatim (class body character-for-character, `__init__` through the final `_process_charts` delegator) plus every module-level import the class needs (`Agent`, `Model`, `InMemoryRAG`, `StockTools`, `DatabaseManager`, `ForecastAgent`, `ClusterContext`, `ForecastResult`, the eight `_*_impl` leaf-module imports, every `deepear.src.prompts.report_agent` function, `re`, `time`, `datetime`/`timedelta`, `SimpleNamespace`, `logger`).
- Collapse `deepear/src/agents/report_agent.py` to a docstring plus `from deepear.src.agents.report.agent import ReportAgent` and `__all__ = ["ReportAgent"]` (34 lines) — the only name any consumer or test ever referenced through this module path.
- Add real re-exports to `deepear/src/agents/report/__init__.py`: `ReportAgent` plus each leaf module's public function (`run_agent_with_retry`, `sanitize_json_chart_blocks`, `build_structured_report`, the six `citations.py` functions, `clean_ticker`/`signal_mentions_ticker`, `extract_forecast_requests`/`build_forecast_map`, `process_charts`, `cluster_signals`), turning the package into a real `__init__.py` (the plan's "no namespace packages" note) rather than the docstring-only placeholder it was.
- Move the five patch targets found in category (b)/(c) to the new namespace:
  - `tests/report_agent_harness.py`: `import deepear.src.agents.report_agent as report_agent_module` → `import deepear.src.agents.report.agent as report_agent_module` (one line; every `monkeypatch.setattr(report_agent_module, ...)` call site in the file is unaffected by the rename since they already go through the variable, not a literal string).
  - `tests/test_report_agent_characterization.py::test_tool_model_defaults_to_model_when_omitted`: string path `"deepear.src.agents.report_agent.Agent"` → `"deepear.src.agents.report.agent.Agent"`.
  - `tests/test_report_agent_citations.py::test_non_incremental_report_generation_normalizes_citations`: same string-path rewrite.
  - `tests/test_report_chart_renderer.py`: both `import deepear.src.agents.report_agent as report_agent_module` sites (inside `TestChartRendererModuleFunctionDirectly`'s two tests) → `import deepear.src.agents.report.agent as report_agent_module`.
  - `tests/test_report_clustering.py`: `import deepear.src.agents.report_agent as report_agent_module` → `import deepear.src.agents.report.agent as report_agent_module`.
  - Accompanying docstring/comment updates in all five files (and `tests/report_agent_harness.py`'s module/function docstrings) explaining the new seam, so the next reader does not reintroduce a patch against the shim.
- Add `tests/test_report_agent_shim.py`: proves old import spellings still resolve to the same object, the shim class *is* the package class (identity, `__module__` check), patching the new namespace intercepts a real `generate_report()` call end-to-end (not just construction), and `deepear.src.agents.report`'s `__init__.py` re-exports resolve to the same objects as their fully-qualified imports.
- No production behavior change anywhere; `ReportAgent`'s class body is byte-for-byte identical, just relocated.

## Capabilities

- None.
