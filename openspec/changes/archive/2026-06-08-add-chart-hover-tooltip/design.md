## Context

The visualizer renders a static SVG equity curve at generation time and the browser JavaScript rerenders the same chart when it runs. Tables and cards now have static fallbacks, but point-level chart inspection needs JavaScript because hover events depend on the browser.

## Goals / Non-Goals

**Goals:**
- Add hover behavior over the equity chart.
- Display nearest-date portfolio and benchmark values/returns in an on-page tooltip.
- Preserve existing static chart content when JavaScript is unavailable.

**Non-Goals:**
- Add external charting libraries.
- Add crosshair dragging, zooming, pan controls, or multi-run comparison.
- Change report artifact schema.

## Decisions

- Use vanilla JavaScript event listeners on the existing SVG element.
  - Rationale: keeps the HTML self-contained and avoids dependency/build complexity.

- Compute the nearest row from mouse x-position rather than adding per-point DOM listeners.
  - Rationale: simpler, cheaper, and works for dense equity curves.

- Keep the static SVG generated by Python as the initial chart.
  - Rationale: users still see the chart if JavaScript does not execute.

## Risks / Trade-offs

- [Risk] Hover only works when browser JavaScript is enabled. → Mitigation: static chart remains visible without JavaScript.
- [Risk] The tooltip is portfolio-level, not per-ticker. → Mitigation: ticker filtering remains table/position based; per-ticker equity curves require different artifacts and can be a later change.
